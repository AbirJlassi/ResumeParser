01

Architecture du pipeline

Couche 0
TextExtractor
Ingestion multi-format. PDF natif via pdfminer, bascule automatique en OCR Tesseract si le texte extrait est inférieur à 50 caractères (PDF scanné). DOCX via python-docx avec extraction des tableaux. Images (JPG, PNG, WEBP, TIFF) via OCR direct. Retourne le texte brut + la méthode utilisée.
Couche 1
RegexExtractor
Extraction déterministe haute précision : email (RFC 5321), téléphone (formats FR/international avec exclusion des faux positifs), LinkedIn et GitHub (URL normalisée). Toujours exécutée en premier — ses résultats priment sur toutes les couches suivantes.
Couche 2
NERExtractor
Reconnaissance d'entités nommées via spaCy. Cibles : PER → nom du candidat (2 mots min, initiaux majuscules), GPE/LOC → localisation, ORG → entreprises et écoles (avec blacklist des faux positifs : Python, Docker, etc.). Charge le meilleur modèle disponible selon une liste de priorité FR → EN.
Couche 3
LLMExtractor
Extraction sémantique via Groq API. Reçoit le texte brut + les champs déjà extraits (couches 1 & 2) comme contexte enrichi dans le prompt — réduit les hallucinations. Structured output prompting avec JSON schema explicite. Troncature intelligente au dernier saut de ligne (limite : 60 000 chars pour tous les modèles à 131k tokens de contexte).
Couche 4
PostProcessor
Fusion avec règles de priorité par champ, validation croisée (valeurs LLM vérifiées dans le texte source), normalisation (lowercase skills, déduplication certifications, strings vides → null). Génère les métadonnées _extraction_meta listant les champs où Regex/NER a écrasé le LLM.

PY cv_evaluator.py Benchmark & reporting
05

Système de métriques

Champs scalaires

Exact Match après normalisation (lowercase + strip). Couvre : full_name, email, phone, location, linkedin, github. Score entre 0 et 1 par champ.

Champs liste

Precision / Recall / F1 sur ensembles normalisés. Couvre : skills, certifications, experiences (clé : title), education (clé : degree).

Agrégation

Moyenne sur l'ensemble du dataset pour chaque champ. Les CVs en erreur reçoivent des scores à zéro sans interrompre le benchmark. Le compteur failed_parses trace les échecs.

06

Benchmark runner

Format ground_truth.json

Chaque entrée du dataset contient un id, soit un champ text (texte inline) soit un champ file (chemin fichier), et un objet expected avec les valeurs attendues. Le mode text appelle directement parse_text() — pas de fichier temporaire.

Sortie console

Ligne par CV : statut (✅ / ⚠️ / ❌), nombre de champs scalaires corrects, F1 moyen des listes, latence. Rapport agrégé en fin de run avec barres de progression ASCII.
